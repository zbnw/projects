# Time Encrypt

Time Encrypt 是一个零依赖、零构建、无服务端存储的一刻钟临时密文工具。页面在每次加解密前读取同源服务器/CDN 返回的 HTTP `Date` 响应头，使用服务器校准时间签发和验证密文；不会使用设备墙上时钟作为协议时间。

## 功能

- 有效期固定为 900 秒，第 899 秒可解密，第 900 秒立即拒绝。
- 固定 129 位显示长度：26 组，每组 4 位，以连字符分组。
- 正文仅包含 `0-9`、`A-Z`、`a-z`，便于复制传递。
- 口令保护模式：PBKDF2-SHA256、HKDF-SHA256、AES-256-GCM。
- 免口令便捷模式：无需口令即可解密，适合低敏感临时内容。
- 纯静态文件，不使用 Cookie、`localStorage`、IndexedDB、统计脚本或外部资源。
- 支持键盘操作、窄屏布局与 `prefers-reduced-motion`。

## 先读安全边界

### 口令保护模式

填写至少 8 个字符的共享口令后使用协议 v4。每条密文都有独立随机 IV，IV 同时参与 PBKDF2 盐；PBKDF2-SHA256 迭代 600,000 次，再由 HKDF-SHA256 按 Unix 分钟隔离 AES-256-GCM 密钥。协议版本和固定域作为附加认证数据参与校验。

口令不会离开页面，但静态前端无法限制离线猜测。请使用 16 个以上随机字符，不要复用登录密码。

### 免口令便捷模式

口令留空时使用协议 v5。接收方不需要输入口令，但这意味着生成密钥所需的信息也必须随密文公开。因此：

> v5 不提供秘密性。任何拿到密文的人都能解开；它只提供固定格式、15 分钟页面验证规则和传输损坏检测。

这不是可以通过前端算法消除的限制。若业务要求“无需用户口令，但仍然保密”，必须由后端保存主密钥并执行解密授权。

### 服务器时间并不等于后端密钥

页面使用同源 `HEAD` 请求的 HTTP `Date` 响应头校时，并以 `performance.now()` 单调推进时间。校时失败时会关闭加解密，不回退到本机时间。这能避免直接修改设备时间的低级绕过，但仍有以下边界：

- 时间由站点托管层/CDN响应提供，不是独立时间证明。
- 用户能修改浏览器里运行的公开代码。
- 已经解密或复制的明文无法远程销毁。
- 高敏感生产系统必须把可信时间、主密钥、授权、速率限制和审计放到后端。

## 协议布局

当前写入协议：v4（口令保护）与 v5（免口令）。v3 仅保留短期只读兼容。

```text
版本 1 字节 | 随机 IV 12 字节 | AES-GCM 输出 64 字节 = 77 字节
```

加密前的固定帧为 48 字节：

```text
签发时间 4 字节 | 明文长度 1 字节 | 明文最多 43 字节 | 随机填充
```

AES-GCM 输出包含 48 字节密文和 16 字节认证标签。77 字节整体转换为固定 104 位 Base62，再按每 4 位分成 26 组，最终显示长度为 129。

## 开发者如何修改

### 品牌和文案

页面文字在 `index.html`，视觉令牌集中在 `styles.css` 顶部的 `:root`。可以修改品牌、色彩和说明，但建议保留：

- 免口令模式“不保密”的显著提示；
- 服务器校时失败时的关闭策略；
- 口令不上传、不保存的事实描述；
- 高敏感用途必须后端化的边界说明。

### 协议参数

协议参数集中在 `crypto-core.mjs` 顶部。修改时遵循以下规则：

1. 修改数据帧、派生域、认证参数或编码时增加协议版本。
2. 新协议必须更换 PBKDF2 盐域、HKDF `info` 和 AES-GCM 认证上下文。
3. 修改 `MAX_AGE_SECONDS` 后同步更新页面、README 和边界测试。
4. 修改 `FRAME_LENGTH` 后重新计算明文上限、二进制长度和 Base62 固定长度。
5. 不要把安全性建立在隐藏算法、混淆 JavaScript 或不公开常量上。
6. 密码实现只放在 `crypto-core.mjs`；界面层只负责传入已校准的服务器时间。

### 接入真正后端

生产版本建议让后端完成以下工作：

1. 使用服务器单调时间签发，返回不可伪造的短期令牌。
2. 把主密钥放在密钥管理系统中，绝不下发到浏览器。
3. 使用随机令牌或标准 AEAD 封装，不把分钟时间本身当作秘密。
4. 为解密端点增加身份验证、授权、速率限制、审计与密钥轮换。
5. 通过 HTTP 响应头下发 CSP、HSTS、`X-Content-Type-Options` 等安全策略。

## 最低验证清单

- v4 正确口令可往返，错误口令被拒绝。
- v5 加密后可在不输入口令时解密，并清楚显示“不保密”。
- 任意篡改被 AES-GCM 认证拒绝。
- 相同明文连续加密产生不同密文。
- 第 899 秒可解密，第 900 秒立即失效。
- 密文为 129 位、26 组、每组 4 位，只含 Base62 和分组连字符。
- 43 个 UTF-8 字节可用，44 字节被拒绝；中文边界正常。
- 核心代码不存在 `Date.now()`，所有协议调用必须显式传入服务器时间。
- 校时失败时提交按钮不可用。
- `app.js` 引用的所有元素 ID 都存在且唯一。
- CSP 只允许同源脚本、样式和校时请求；没有第三方网络请求。
